RIPA_SGI es un sistema de gestión de inventario diseñado para optimizar el control y seguimiento de productos y ventas en una panadería. Este sistema facilita a los empleados y administradores monitorear y analizar las transacciones diarias, facilitando así el seguimiento de las ventas en tiempo real y generando informes detallados para la toma de decisiones estratégicas.
RIPA_SGI cuenta con un modulo de gestión de inventario que tiene como características principales:

-Control preciso
Seguimiento del stock de cada
producto, incluyendo fechas de
vencimiento y lotes.

-Alerta de stock
Notificaciones automáticas cuando
el inventario está bajo, evitando
faltantes de productos.

-Gestión de proveedores
Registro de proveedores, precios de
compra, facturas y pedidos.

RIPA_SGI cuenta con un modulo de gestión de ventas que tiene como características principales:
-Punto de venta
Facilita la realización de ventas, con registro de
productos, cantidades y precios.

-Control de pedido
Gestión de pedidos anticipados, seguimiento del
estado y emisión de facturas.

-Reporte de venta
Análisis de ventas por producto, fecha, cliente, mesa y
otras variables clave.

Por ultimo RIPA_SGI incluye un módulo de gestión de usuarios que cuenta con las siguientes características:

-Roles de usuarios
Define roles y permisos controlando las acciones que cada usuario puede realizar

-Seguridad de acceso
Control de acceso con user y password, también para cada empleado hay un código especifico que permite saber quien realiza cada una de los movimientos, evitando accesos no autorizados

-Gestión de usuarios
Agrega, edita y elimina usuarios con facilidad

Guía para leer documentación RIPA_SGI

En la carpeta del proyecto se encuentra toda la documentación que se ha empleado para llevar a cabo este proyecto. En la carpeta material de apoyo se puede encontrar todos los materiales gráficos y documentos empleados para la estructura del proyecto, se recomienda leer en el siguiente orden:

-Diapositiva RIPA: En este documento se encuentra la idea inicial de lo que se busca para RIPA_SGI empleando varias estrategias como generar una pregunta problema y objetivos para darle solución a la problemática planteada.

-Plantilla ERS: En la plantilla ERS se busca seguir con la idea de la diapositiva anterior, con la diferencia de que en este documento se profundiza mas cada uno de los aspectos planteados.

-Diagrama de clases: En el diagrama de clases de encuentra cada una de las clases que se emplean para la estructura del código Java, también se muestran las relaciones que tienen cada una de estas para la base de datos en MySQLworkbench

-Diagrama de casos de usos: En este diagrama sirve para especificar la comunicación y el comportamiento de un sistema mediante su interacción con los actores y/u otros sistemas.

-Diagrama UML: En el diagrama UML se encuentra cada uno de los métodos que se emplean para cada una de las clases
